## 📚 Теоретический материал

### 1.1 С чего всё начинается — зачем нужна аналитика

**Аналитика данных — это способ принимать правильные решения, основываясь на фактах, а не на догадках.**

#### Примеры из жизни:

**Плохое решение (без данных):**
- "Мне кажется, клиенты любят красный цвет. Сделаем всю упаковку красной."
- Результат: продажи упали на 30%

**Хорошее решение (с данными):**
- Опросили 500 клиентов → 68% предпочитают синий цвет → сделали упаковку синей
- Результат: продажи выросли на 25%

#### Где используется аналитика:
- **В магазине**: какие товары покупают чаще всего?
- **В банке**: кому можно давать кредит, а кому нет?
- **В интернете**: какая реклама работает лучше?
- **В HR**: почему сотрудники увольняются?

### 1.2 Данные вокруг нас — учимся их замечать

**Данные — это записанная информация, которую можно посчитать или сравнить.**

#### Данные в повседневной жизни:

**Дома:**
- Список покупок (что, сколько, цена)
- Счета за коммунальные услуги (месяц, сумма, тип услуги)
- Рецепты (ингредиенты, количество, время готовки)

**На работе:**
- Список клиентов (имя, контакты, дата регистрации)
- Продажи (товар, количество, дата, цена)
- Сотрудники (должность, зарплата, стаж)

#### Типы данных (без сложных терминов):

**Цифры** — то, что можно посчитать:
- Возраст: 25, 34, 41
- Цена: 1500, 890, 2300
- Количество: 5, 12, 0

**Категории** — то, что можно разделить на группы:
- Пол: мужской, женский
- Город: Москва, СПб, Казань
- Оценка: отлично, хорошо, плохо

**Даты и время** — когда что-то происходило:
- 15.03.2024, 09:30, вторник

### 1.3 Почему аналитика — это не сложная математика

**Главный секрет: в аналитике данных 80% работы — это здравый смысл и только 20% — технические навыки.**

#### Развеиваем мифы:

**МИФ**: "Нужно знать высшую математику"  
**ПРАВДА**: Достаточно уметь складывать, вычитать, находить среднее значение

**МИФ**: "Нужно программировать"  
**ПРАВДА**: Большинство задач решается в Excel или готовых программах

**МИФ**: "Это только для технических специальностей"  
**ПРАВДА**: Лучшие аналитики часто приходят из экономики, социологии, психологии

#### Что действительно важно:
- **Любопытство**: "А почему так происходит?"
- **Логика**: "Если A, то должно быть B"
- **Внимательность**: "Что-то здесь не так..."
- **Умение объяснять**: "Как рассказать это простыми словами?"

### 1.4 Знакомство с инструментами — наш рабочий набор

**Наш стек инструментов — как набор отвёрток у мастера. Каждый инструмент для своих задач.**

#### Excel — швейцарский нож аналитика
**Для чего используем:**
- Простые расчёты (сумма, среднее, проценты)
- Создание таблиц и простых графиков
- Быстрый анализ небольших данных (до 10000 строк)

**Аналогия:** как калькулятор, только умнее

#### Power Query — сборщик данных
**Для чего используем:**
- Подключение к разным источникам данных
- Очистка и подготовка данных
- Объединение нескольких таблиц в одну

**Аналогия:** как помощник, который собирает информацию из разных мест и приводит в порядок

#### Power Pivot — строитель моделей
**Для чего используем:**
- Работа с большими объёмами данных (миллионы строк)
- Создание связей между таблицами
- Сложные расчёты одной формулой

**Аналогия:** как архитектор, который проектирует здание из данных

#### Power BI — рассказчик историй
**Для чего используем:**
- Создание красивых дашбордов
- Интерактивные графики
- Презентация результатов руководству

**Аналогия:** как художник, который превращает цифры в понятные картины

#### Python — автоматизатор
**Для чего используем:**
- Сложные статистические расчёты
- Машинное обучение и прогнозы
- Автоматизация рутинных задач

**Аналогия:** как робот, который делает сложную работу за вас

#### SQL — кладовщик данных
**Для чего используем:**
- Извлечение данных из больших баз
- Быстрые запросы к миллионам записей
- Подготовка данных для анализа

**Аналогия:** как библиотекарь, который мгновенно найдёт нужную книгу среди миллионов

### Python: первая программа (5 минут)

**1. Откройте Jupyter Notebook**
- Пуск → Anaconda Navigator → Jupyter Notebook

**2. Создайте новый файл:**
- New → Python 3

**3. Напишите первую программу:**
```python
# Список количества проданных чашек кофе за каждый день недели
coffee_cups = [45, 52, 38, 41, 67, 89, 73]

# Список названий дней недели, соответствующих индексам в списке coffee_cups
days = ["Понедельник", "Вторник", "Среда", "Четверг", "Пятница", "Суббота", "Воскресенье"]

# Вычисляем среднее количество чашек в день:
# sum(coffee_cups) — общая сумма проданных чашек,
# len(coffee_cups) — количество дней (7)
average = sum(coffee_cups) / len(coffee_cups)

# Выводим рассчитанное среднее значение
print(f"Среднее количество чашек в день: {average}")

# Находим индекс дня с максимальными продажами чашек:
# max(coffee_cups) — наибольшее значение в списке,
# coffee_cups.index(...) — индекс этого значения
best_day_index = coffee_cups.index(max(coffee_cups))

# Выводим название лучшего дня по индексу best_day_index
print(f"Лучший день: {days[best_day_index]}")
```

**4. Запустите программу:**
- Нажмите Shift + Enter
